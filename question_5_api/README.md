# Question 5 — Clinical Trial Data API

![Python](https://img.shields.io/badge/Python-3.11-blue?logo=python)
![FastAPI](https://img.shields.io/badge/FastAPI-0.110-green?logo=fastapi)
![Pandas](https://img.shields.io/badge/Pandas-2.0-150458?logo=pandas)
![License](https://img.shields.io/badge/License-MIT-yellow)

A RESTful API built with **FastAPI** that serves Adverse Event (AE) clinical trial data exported from `pharmaverseadam::adae`. It supports dynamic cohort filtering and computes a per-subject weighted Safety Risk Score.

---

## Project Structure

```
question_5_api/
├── main.py             # FastAPI application (3 endpoints)
├── prepare_data.R      # One-shot R script to export adae.csv

├── data/
│   └── adae.csv        # Generated by prepare_data.R (not tracked in git)
├── tests/

└── README.md
```

## Setup

### Prerequisites
- Python 3.12+
- R 4.2+ with `pharmaverseadam` installed
- A virtual environment manager (venv or conda)

### Step 1 — Generate the input data (R)

Run the following from the `question_5_api/` directory to export the AE dataset as a CSV:

```r
Rscript prepare_data.R
```

This will create `data/adae.csv` containing the required columns (`USUBJID`, `AESEV`, `ACTARM`) from `pharmaverseadam::adae`.

### Step 2 — Create a virtual environment

**Option A — venv (PowerShell on Windows)**
```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

**Option B — conda**
```bash
conda create -n q5_api python=3.12 -y
conda activate q5_api
```

### Step 3 — Install dependencies

```bash
pip install -r requirements.txt
```

### Step 4 — Start the API

```bash
uvicorn main:app --reload
```

The API will be available at **http://127.0.0.1:8000**

> **Tip:** Visit **http://127.0.0.1:8000/docs** for the interactive Swagger UI where you can test all endpoints directly in the browser.

## Endpoints

### `GET /`
Health check — confirms the API is running.

```bash
curl http://127.0.0.1:8000/
```

**Response:**
```json
{"message": "Clinical Trial Data API is running"}
```

---

### `POST /ae-query`
Dynamically filter the AE dataset by severity and/or treatment arm. All fields are optional — omitted fields return all records for that dimension.

```bash
curl -X POST http://127.0.0.1:8000/ae-query \
     -H "Content-Type: application/json" \
     -d '{"severity": ["MILD", "MODERATE"], "treatment_arm": "Placebo"}'
```

**Request body:**
| Field | Type | Required | Description |
|---|---|---|---|
| `severity` | `list[str]` | No | e.g. `["MILD", "MODERATE"]` |
| `treatment_arm` | `str` | No | e.g. `"Placebo"` |

**Response:**
```json
{
  "count": 213,
  "subjects": ["01-701-1015", "01-701-1023", "..."]
}
```

---

### `GET /subject-risk/{subject_id}`
Compute a weighted Safety Risk Score for a given subject based on their AE severities.

```bash
curl http://127.0.0.1:8000/subject-risk/01-701-1015
```

**Scoring logic:**

| AESEV | Points |
|---|---|
| MILD | 1 |
| MODERATE | 3 |
| SEVERE | 5 |

**Risk categories:**

| Score | Category |
|---|---|
| < 5 | Low |
| 5 – 14 | Medium |
| ≥ 15 | High |

**Response:**
```json
{
  "subject_id": "01-701-1015",
  "risk_score": 8,
  "risk_category": "Medium"
}
```

**Error response (404)** — returned if the subject ID does not exist:
```json
{
  "detail": "Subject '01-701-9999' not found in AE dataset."
}
```

## Design Notes

- **Single CSV load at startup** — the dataset is read once into memory when the app starts. Since the API is read-only, a process-wide DataFrame is efficient and keeps each request O(rows).
- **Case-insensitive matching** — `AESEV` and `ACTARM` are compared without case sensitivity so the API accepts `"mild"` and `"Placebo"` alike, while preserving original values in responses.
- **Unknown severities contribute 0** — AEs with missing or unrecognised severity values are safely handled without crashing.
- **Fail-fast validation** — missing CSV or missing required columns raise a clear error at startup rather than failing silently on the first request.
- **Self-documenting** — Pydantic models power the OpenAPI schema at `/docs`, so the API documents itself automatically.
